2021 Accelerate State of DevOps Report 日本語要約

原題: Accelerate: State of DevOps 2021 | 発行: Google Cloud DevOps Research and Assessment（DORA） | 原文言語: 日本語（公式日本語版） | 要約作成日: 2026-06-04

① レポート概要

本レポートは、Google Cloud の DevOps Research and Assessment（DORA）チームが作成した 2021 年版の「Accelerate State of DevOps」レポートである。32,000 を超える全世界の専門家に対する 7 年間の調査と、そこから得られたデータを基盤としており、2021 年は全世界の各種業界に属する 1,200 人の現役専門家が調査に回答した。レポートはソフトウェア デリバリー、運用、組織的なパフォーマンスを推進する能力とプラクティスを、厳密な統計的技法（クラスタ分析、構造方程式モデリングなど）を用いて分析し、テクノロジーを開発・提供する最も効果的で効率的な方法についてのデータドリブンな分析情報を提示する。構成は、1章 エグゼクティブサマリー、2章 比較方法、3章 改善のための提案（クラウド、SRE と DevOps、ドキュメント、セキュリティ、技術面の DevOps 機能、COVID-19、文化）、4章 調査対象者、5章 最後に、以降は謝辞・作成者・手法・関連情報となっている。

② エグゼクティブサマリー

調査により、優れたソフトウェア デリバリーと運用のパフォーマンスが、技術変革における組織的なパフォーマンスを促進することがあらためて実証された。チームが自分たちを業界と比較してベンチマークできるよう、クラスタ分析により「低・中・高・エリート」というパフォーマンス カテゴリを形成している。チームは自らの現在位置を認識した上で、予測分析の発見事項を使って主要な結果を改善し、相対的な地位を向上させるための能力とプラクティスの目標を設定できる。

2021 年版では特に、信頼性のターゲットを満たすこと、ソフトウェアのサプライ チェーン全体にわたってセキュリティを統合すること、高品質の内部ドキュメントを作成すること、クラウドのポテンシャルを最大限に活用することの重要性が強調されている。また、肯定的なチーム文化が、COVID-19 パンデミックの結果としてリモートで勤務することの影響を軽減できるかどうかも探求された。意味のある改善には継続的な改善の原理の採用が必要であり、ベンチマークで現状を測定し、制約を識別し、その制約を緩和する改善を試みることが推奨されている。

③ 主な調査結果

1. 最高パフォーマンスのグループは成長し続け、水準はさらに上がっている。 エリート パフォーマンスのグループは全体の 26% を占めるまでに拡大し、本番環境への変更のリードタイムも短縮した（2019 年の 1 日未満から 2021 年は 1 時間未満へ）。調査開始以降初めて、回答者の 2/3 が高およびエリートのパフォーマンスを持つチームとなった。今年初めて、変更時の障害率を最小化したのはエリートのグループだけとなった。
2. SRE と DevOps は相補的な理念である。 サイト信頼性エンジニアリング（SRE）の最新の運用プラクティスを活用しているチームは、より高い運用パフォーマンスを報告している。これらのプラクティスに優れているチームは、優れた SDO パフォーマンスを報告する可能性が 1.4 倍、良好なビジネス成果を報告する可能性が 1.8 倍高い。回答者の 52% が SRE プラクティスをある程度使用していたが、エリート回答者でもすべての SRE プラクティスを完全実装していたのはわずか 10% だった。
3. より多くのチームがクラウドを活用し、大きな恩恵を受けている。 パブリック クラウド利用者は 56% で 2019 年より 5% 増加し、21% は複数のパブリック クラウドにデプロイしている。ハイブリッドやマルチクラウドの利用者は、組織のパフォーマンス目標を超える割合が 1.6 倍、SDO パフォーマンスに優れる可能性が 1.4 倍高い。エリートのグループは NIST の本質的なクラウド特性 5 つすべてを満たしている割合が 3.5 倍高い。
4. 安全なソフトウェア サプライ チェーンは不可欠であり、パフォーマンスも引き上げる。 2020 年には 220 億レコードを超える機密情報が漏えいし、SolarWinds や Codecov のサプライ チェーン攻撃も発生した。開発プロセス全体にセキュリティを統合したチームは、ソフトウェアを迅速・高信頼・安全に提供しており、組織目標を満たす、または超える割合も 1.6 倍高い。
5. 良質なドキュメントの作成は DevOps を効果的に機能させるための基礎である。 今回初めて内部ドキュメントの品質を測定した。良い品質のドキュメントを保有する回答者は約 25% で、高品質のドキュメントを持つチームは SDO パフォーマンスに優れる割合が 2.4 倍、セキュリティ プラクティスを実装する割合が 3.8 倍、信頼性目標を満たす割合が 2.4 倍、SRE プラクティスを実装する割合が 3.5 倍、クラウドを完全活用する割合が 2.5 倍だった。
6. 肯定的なチーム文化は困難な状況での燃え尽き症候群を軽減する。 ジェネレーティブ（Westrum の類型論におけるパフォーマンス指向）な文化を持ち、メンバーが帰属と受け入れを感じる包括的なチームは、COVID-19 パンデミックの最中に燃え尽き症候群を経験する割合が半分だった。エリートで信頼性目標を満たしたグループは、ジェネレーティブな文化を持つ割合が低パフォーマンスのグループの 2.9 倍だった。
7. 業界全体が加速し続けている。 パンデミックの最中にも SDO パフォーマンスの低下傾向は見られず、GitHub の調査でも 2020 年を通して開発者のアクティビティの増加が示された。業界はパンデミックの結果というよりも、パンデミックにもかかわらず加速し続けていると考えられる。

⑤ 章ごとの要点

2 章 比較方法（ソフトウェア デリバリーと運用のパフォーマンス）

4 つのソフトウェア デリバリー指標（デプロイの頻度、変更のリードタイム、サービス復旧時間、変更時の障害率）はスループットと安定性を測定し、クラスタ分析によりエリート・高・中・低の 4 プロファイルが識別された。これらの指標に運用パフォーマンス（信頼性）を加えたものを SDO（ソフトウェア デリバリーと運用）パフォーマンスと呼ぶ。デリバリー パフォーマンスのすべてのカテゴリにおいて、信頼性目標を満たした / 超えたチームは複数の結果にわたって大きな利益が見られた。エリートの割合は 26% に達し、回答者の 2/3 が高・エリートとなって業界全体の加速が示された。

3 章 改善のための提案 — クラウド

パブリック クラウド利用は 56%（2019 年比 +5%）、マルチクラウドは 21%、ハイブリッド クラウドは 34%、プライベート クラウドは 29%、クラウド未使用（データセンター / オンプレミス）は 21%。複数プロバイダ利用の最大の理由は「各プロバイダ固有の利点の活用」（26%）で、次が「可用性」（22%）、「障害復旧」（17%）、「法令遵守」（13%）。重要なのはクラウドを使うこと自体ではなく「どのように実装するか」であり、NIST の 5 つの本質的特性（オンデマンドのセルフサービス、広範なネットワーク アクセス、リソースのプーリング、迅速な弾力性、メジャード サービス）の利用は 2019 年から 14〜19% 増加し、特に「迅速な弾力性」が最も増加した（+18%）。

3 章 — SRE と DevOps

SRE と DevOps は競合ではなく補完性の高い理念である。SLI / SLO 指標フレームワークとエラー バジェット、手作業とアラートの自動化による削減、インシデント対応プロトコルと演習、「信頼性のシフトレフト」などの行動を評価した。SRE プラクティスの採用が進むほど燃え尽き症候群を経験する可能性が低く、SRE で信頼性目標を満たしたチームはコード作成により多くの時間を費やしていた。どの SDO パフォーマンス レベルのチームも SRE プラクティスの利用拡大から恩恵を受けられる。

3 章 — ドキュメント

初めて内部ドキュメント（マニュアル、README、コードコメントなど）の品質を測定した。品質を高めるプラクティスとして、重要なユースケースの文書化、更新・編集の明確なガイドライン、オーナー（所有権）の定義、ソフトウェア開発プロセスの一部としてのドキュメント作成、パフォーマンス レビューや昇進時のドキュメント作業の評価が挙げられた。トレーニング、コードサンプルの自動テスト、スタイルガイドなども品質を支える。

3 章 — セキュリティ

セキュリティは後付けではなく、ソフトウェア開発プロセス全体に組み込む必要がある（シフトレフト）。業界は事後対応的・予防的な手法から診断的な手法へ移行し、「自分たちのシステムはすでに侵害されている」と想定してサプライ チェーンにセキュリティを構築すべきである。具体的プラクティスの採用率は、InfoSec の早期かつ頻繁な関与 63%、セキュリティのレビュー 60%、セキュリティのテスト 58%、すべてのフェーズへのセキュリティ レビューの統合 54%、事前承認済みコードのビルド 49%。高品質ドキュメントを持つチームはセキュリティを開発プロセス全体に統合している割合が 3.8 倍だった。

3 章 — 技術面の DevOps 機能

疎結合アーキテクチャ、トランクベース開発、継続的テスト、継続的インテグレーション、オープンソース テクノロジー、モニタリングとオブザーバビリティ、データベースの変更管理、デプロイの自動化を測定した。中でも疎結合アーキテクチャと継続的テストが継続的デリバリーに最も大きな影響を持つ。トランクベース開発では作業を最低 1 日 1 回、できれば複数回トランクに結合すべきで、継続的インテグレーションと同時に取り入れることで最大の効果を生む。優れたオブザーバビリティを持つチームはコーディングに多くの時間を費やしていることも示された。

3 章 — COVID-19 と文化

回答者の 89% がパンデミックのため在宅勤務しており、パンデミック前から在宅勤務経験があったのは 20% にすぎない。それでも SDO パフォーマンスの低下傾向は見られなかった。燃え尽き症候群を左右する最大の要因は文化であり、ジェネレーティブな文化と帰属・受け入れの感覚を持つチームは燃え尽き症候群の経験が半分だった。Westrum の類型論（パワー指向、ルール指向、パフォーマンス指向)に基づき、情報の流れ・信頼・イノベーション・リスク共有を最適化する文化が高い SDO パフォーマンスを予測する。心理的安全性や帰属・インクルーシブネスを重視する組織は燃え尽き症候群のレベルが低い。

4 章 調査対象者

回答者は男性 83%、女性 12%、ノンバイナリー 1%。チームの女性比率は中央値 25%（2019 年の 16% から回復）。障がいを持つ人員は 9%、社会的地位の低いグループは 17%（2019 年は 13.7%）。41% は 16 年以上の経験を持ち、85% 以上が 6 年以上の経験を持つ。部門は開発 / エンジニアリング 23%、DevOps と SRE 21%、マネージャー 18%。業種はテクノロジー 33%、金融サービス 14%、小売 9% など。22% は従業員 10,000 人超の企業に勤務。62% は 10 人以下のチームに所属。地域は北米 39%（2019 年の 50% から減少）、欧州 32%、アジア 13%。デプロイ ターゲットはコンテナ 64%、VM 49%、FaaS 48%、ベアメタル サーバー 30%、PaaS 30%。

5 章 最後に / 8 章 手法

7 年間の調査から DevOps が組織にもたらす利益は依然として明白で、組織は年々加速し改善され続けている。「ソリューションの構造に人員を適応させるのではなく、ソリューションの能力を利用する人員に適した構造を作り上げる」ことの重要性が強調された。手法面では、理論に基づく推論予測設計、雪だるま式サンプリング、潜在クラス分析（ベイズ情報量基準でクラスタ数を決定）、探索的因子分析・主成分分析、部分的最小二乗（PLS）による構造方程式モデリングが用いられた。

⑥ 実践への示唆

  • 継続的な改善の原理を採用する。ベンチマークで現状を測定し、調査された能力に基づいて制約を識別し、その制約を緩和する改善を試みる。勝利と失敗のどちらからも教訓を得て有意義な行動につなげる。
  • デリバリーと運用の両方を重視する。デリバリーのパフォーマンスと運用のパフォーマンス（信頼性）の両方を重視するチームが、最も高い組織的パフォーマンスを報告している。SLI / SLO とエラー バジェットなどの SRE プラクティスの採用を拡大する。エリートでも完全実装は 10% であり、成長の余地は大きい。
  • クラウドは「使い方」が重要。NIST の 5 つの本質的特性を満たす形でクラウドを実装し、各プロバイダ固有の利点を活用する。
  • セキュリティをライフサイクル全体に統合する。テストへのセキュリティの組み込み、全フェーズでのセキュリティ レビュー、事前承認済みコードのビルド、InfoSec の早期かつ頻繁な関与を実践する。
  • ドキュメントに投資する。ユースケースの文書化、更新ガイドライン、所有権の定義、開発プロセスへの組み込み、評価制度での認知により品質を高める。ドキュメント品質のどのような改善もパフォーマンスに肯定的かつ直接的な影響を持つ。
  • 技術プラクティスを組み合わせて導入する。疎結合アーキテクチャと継続的テストが最も影響が大きい。トランクベース開発と継続的インテグレーション、さらにデプロイの自動化を一緒に使用すればより大きな改善が期待できる。データベース変更は更新前にチームでレビューする。DevOps 変革ではオープンソース ソフトウェアの利用拡大が推奨される。
  • 文化に関連する問題への対処に力を注ぐ。ジェネレーティブで、帰属とインクルーシブネスを重視する文化は、高い SDO・組織パフォーマンスと低い燃え尽き症候群を予測する。文化は流動的で常に変化し続けているため、変えていくことが可能である。
