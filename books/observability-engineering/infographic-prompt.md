# 一枚絵インフォグラフィック生成プロンプト（オブザーバビリティ・エンジニアリング）

`books/observability-engineering/index.html` の内容を、手描き風ホワイトボード解説イラスト1枚にまとめるための画像生成プロンプト。出力先モデルは Nano Banana Pro（Gemini 3 Pro Image）／ GPT Image 2 を想定。

## 使い方のコツ（重要）

- 下のプロンプトの **「描く文字」は一字一句そのまま** 渡す。抽象指示だと英語混入・文字化けの原因。
- 末尾の **「指定した文字以外は描かない」** 指示を消さない。
- 「日本語で」などの余計な語は足さない。
- 文字が潰れる場合は、ゾーンを減らすか「コンパクト版」を使う。

> 注：図中の文言はすべて index.html の内容（本書の主張）に基づく。創作の主張・比喩は入れていない。

```text

蛍光ペンの下線、やわらかいパステル配色、手書き風の日本語フォント、かわいい丸いロボットの
マスコット。横長16:9、情報は左から右・上から下へ流れる構成、余白をとって読みやすく明るいトーン。

【上部：タイトル帯】
大きな手書き見出し「オブザーバビリティ・エンジニアリング」、その下に小さく「要点まとめ 全5部22章」。
右上のクリップ留め付箋に「オライリー・ジャパン」。

【中央：主役】
丸いロボットが虫眼鏡を持ち、箱型のシステムに「？」マークを投げかけている絵。
見出し「オブザーバビリティとは？」。
システムから外へ向かう3本の矢印に、それぞれ蛍光ペンの下線つきラベルを置く。
表示文字：システムの内部状態 ／ 外部出力（テレメトリー） ／ 任意の質問で説明できる

【中段左右：対比カード（VS）】
2枚の角丸カードを並べ、間に小さな「VS」。
左カード見出し「モニタリング」、その下「既知の未知」。
右カード見出し「オブザーバビリティ」、その下「未知の未知」。
表示文字：モニタリング ／ 既知の未知 ／ VS ／ オブザーバビリティ ／ 未知の未知

【下部：ロードマップ（道に立て札5本・左から右へ）】
くねった道に番号つきの立て札を5本並べる。色は順に青緑・緑・茶・紫・赤紫。
表示文字：
第Ⅰ部 オブザーバビリティへの道
第Ⅱ部 オブザーバビリティの基礎
第Ⅲ部 チームのためのオブザーバビリティ
第Ⅳ部 大規模なオブザーバビリティ
第Ⅴ部 オブザーバビリティの文化を拡大する

【最下部：キャプション】
細い手書き文字で1行。
表示文字：モニタリングは既知の未知、オブザーバビリティは未知の未知に効く

画像に描く文字は上記で挙げたものだけ。指定した文字以外の文字・ラベル・英単語は描かない。
漢字・ひらがな・カタカナを正確に、文字化けや誤字なく描く。
```

## GPT Image 2 版（制作ブリーフ形式）

```text
オブザーバビリティ・エンジニアリングという本の要点をまとめた、手描きホワイトボード解説風の
インフォグラフィックを横長1枚で作成してください。全体は手描きのマーカースケッチ調で、線は
不揃いにラフに、クリーム色の紙背景、ラフな角丸枠、蛍光ペンの下線、パステル配色、手書き風の
日本語フォント、かわいい丸ロボットのマスコット、明るく親しみやすいトーン。情報は上から下・
左から右へ流れる構成で、余白をとって読みやすく。

上部に手書き見出し「オブザーバビリティ・エンジニアリング」、その下に小さく「要点まとめ 全5部22章」、
右上のクリップ留め付箋に「オライリー・ジャパン」。

中央を主役にして、丸ロボットが虫眼鏡を持ち箱型のシステムに「？」を投げる絵を描き、見出し
「オブザーバビリティとは？」を置く。システムから外へ伸びる3本の矢印に、蛍光ペンの下線つきで
「システムの内部状態」「外部出力（テレメトリー）」「任意の質問で説明できる」と書く。

中段に角丸カードを2枚並べて間に「VS」を置き、左カードに「モニタリング」「既知の未知」、
右カードに「オブザーバビリティ」「未知の未知」と書く。

下部にくねった道と番号つき立て札5本を左から右へ並べ、順に「第Ⅰ部 オブザーバビリティへの道」
「第Ⅱ部 オブザーバビリティの基礎」「第Ⅲ部 チームのためのオブザーバビリティ」
「第Ⅳ部 大規模なオブザーバビリティ」「第Ⅴ部 オブザーバビリティの文化を拡大する」と書く。
最下部に細い手書き文字で「モニタリングは既知の未知、オブザーバビリティは未知の未知に効く」。

画像に描く文字は、ここで挙げた日本語の文言だけにしてください。指定していない文字・英単語は
描かないでください。漢字・ひらがな・カタカナは正確に、文字化けや誤字なく描いてください。
```

API で比率を指定する例：`size: "1536x1024"`（横長）, `quality: "high"`。

## コンパクト版（文字を減らして確実に読ませたいとき）

ロードマップ5本を省き、中央の定義と対比だけに絞った版。文字潰れを避けたいときに使う。

```text
オブザーバビリティ・エンジニアリングの要点を、手描きマーカースケッチ調のインフォグラフィックで
横長1枚にまとめてください。クリーム色の紙背景、ラフな角丸枠、蛍光ペンの下線、パステル配色、
手書き風日本語フォント、かわいい丸ロボット。

上部に見出し「オブザーバビリティとは？」。中央に丸ロボットが虫眼鏡で箱型システムに「？」を投げる絵と、
3本の矢印ラベル「システムの内部状態」「外部出力（テレメトリー）」「任意の質問で説明できる」。
下に2枚のカードを「VS」で並べ、左「モニタリング／既知の未知」、右「オブザーバビリティ／未知の未知」。

描く文字は上記の日本語だけ。指定外の文字・英単語は描かない。文字化けなく正確に描いてください。
```

## 配色メモ（index.html のアクセント色を引き継ぐ）

- タイトル・中央＝青緑 `#1f6f8c`
- 第Ⅰ部＝青緑 `#1f6f8c` ／ 第Ⅱ部＝緑 `#1f7a6b` ／ 第Ⅲ部＝茶 `#8c5a3a` ／ 第Ⅳ部＝紫 `#6b3a8c` ／ 第Ⅴ部＝赤紫 `#8c3a63`
- 対比カードは、モニタリング＝灰青系（既知）、オブザーバビリティ＝青緑系（強調）で差をつける。
- モデルは16進に厳密一致しないが、方向づけにはなる。

## 仕上げの調整

- 文字が崩れた箇所は、その部分だけ**部分再生成（インペイント）**で直す。GPT は会話で「○○の文字を直して」と修正できる。
- 立て札5本の文字が潰れるなら、ロードマップを省いた**コンパクト版**を使う。
- 章扉などにシリーズ化する場合は、本プロンプトのスタイル指定を固定し、中央ゾーンのテーマだけ差し替える（よい1枚を Nano Banana Pro の参照画像にすると絵柄を保てる）。
