# 一枚絵インフォグラフィック生成プロンプト（GPT Image 2 向け）

`index.html` の内容を、手描き風ホワイトボード解説イラスト1枚にまとめるための画像生成プロンプト。
想定モデル：**GPT Image 2（OpenAI）**。ChatGPT の画像生成、または API（`gpt-image-2`）で使う。
※ Nano Banana Pro 版は `infographic-prompt.md` を参照。

## GPT Image 2 で使うときのコツ（Nano Banana との違い）

- GPT Image 2 は**レイアウトを推論して整える**ので、箇条書きの設計図より**一段落の制作ブリーフ**として渡すと馴染みやすい。
- **比率・解像度はプロンプトではなくパラメータ**で指定する（landscape `1536x1024` など）。プロンプトにも「横長」と添える程度でよい。
- 文字描画は強いが、**入れる文字は具体的に列挙**するのが安全。生成後に**チャットで対話的に修正**できる（「右下のロボットのラベルを○○に直して」など）。
- ChatGPT 上なら、崩れた文字や配置を**会話で部分修正**していくのが最も速い。
- 仕上げが端正になりがちなので、**「手描き・マーカー・ラフ」を強めに指定**して親しみやすさを出す。

> 注：図中の文言はすべて本書（`index.html`）の内容に基づく。著者にない主張・創作の比喩は入れていない。

## メインプロンプト（全部入り・制作ブリーフ形式）

```text
書籍の要点をまとめた、手描きホワイトボード解説風のインフォグラフィックを横長1枚で作成してください。
全体は手描きのマーカースケッチ調で、クリーム色の紙の背景、ラフな角丸の枠線、蛍光ペンの下線、
やわらかいパステル配色、手書き風の日本語フォント、かわいい丸いロボットのマスコットを散りばめた、
明るく親しみやすいトーン。情報は左から右へ流れる構成で、余白をとって読みやすく。

上部にタイトル帯を置き、大きな手書き見出しで「The DevOps ハンドブック」、その下に小さく
「理論・原則・実践のすべて」。右上の付箋に「要点まとめ」、左上のクリップ留めメモに
「著者：ジーン・キム 他 ／ 訳：長尾高弘 ／ 日経BP ／ 2017年」。

中央を主役にして、見出し「3つの道（The Three Ways）」を置く。左端に「開発（Dev）」のロボット、
右端に「運用（Ops）」のロボットを向かい合わせに描き、三つの流れを矢印で表現する。
青の右向き矢印に「第一の道：フロー」「開発→運用へ 速く流す」、
赤紫の左向き矢印に「第二の道：フィードバック」「問題を早く見つけ 源流で直す」、
全体を囲む緑の循環矢印に「第三の道：継続的な学習と実験」「失敗から学び 改善を続ける」。

左下に小さな対比図を置き、暗い渦巻きに「負のスパイラル」「Dev と Ops の対立」、
そこから明るい矢印で「DevOps が断ち切る」へつなぐ。

右側に見出し「高パフォーマンスの成果」を置き、マーカー枠の小カードを4枚並べる：
「デプロイ頻度 ↑」「変更リードタイム ↓」「MTTR ↓」「変更失敗率 ↓」。その下に笑顔アイコンと
「働く人の幸福」。

下部に横一列のロードマップを置き、番号付きの立て札を6つ並べる：
「第1部 3つの道」「第2部 どこから始めるか」「第3部 フローの技術的実践」
「第4部 フィードバックの技術的実践」「第5部 継続的な学習と実験」「第6部 情報セキュリティの統合」。

右下に小さなロボット4体を置き、吹き出しで「開発者」「運用・SRE」「マネージャー」「情報セキュリティ」。

画像に描く文字は、ここで挙げた日本語の文言だけにしてください。指定していない文字・ラベル・英単語は
描かないでください。漢字・ひらがな・カタカナは正確に、文字化けや誤字なく描いてください。
```

API で size を指定する場合の例：`size: "1536x1024"`（横長）, `quality: "high"`。

## 仕上げの調整（GPT Image 2）

- ChatGPT 上なら、生成後に「○○の文字が崩れている、直して」「ロードマップを省いてシンプルに」と**会話で反復修正**するのが最速。
- 端正になりすぎたら「もっとラフな手描き感を強く」「線を不揃いに」と追記する。
- 章扉用に各部だけの図解が欲しいときは、中央ゾーンの説明を各部のテーマに差し替える。
